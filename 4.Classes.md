# Classes
## Why we need classes?
Class is blueprint, combined data and functions into one unit.
###
class class_name{...};

## Basics:
• contains declaration of variables and functions
###
• funcs and vars collectively called "class members"
###
• private(protected data) and public(easy access data) sections typed data, know as "visibility labels"

## C++ Class Definitions:
we define objects(variables) and methods(functions)

## Four Pillars of OOP
### • polymorphism  - same func name, different behaviour
### • encapsulation - grouping information into single block of code
### • abstraction   - hiding and securing information
### • inheritance   - inheriting features of other classes by another classes

## Visibility labels:
• Private members can be accessed only within a class, otherwise will raise compilation error
###
• By default class is 'private'
###
• If you change 'public' to 'private', to get objects  will raise a complation error

## Defining Member Functions:
• Outside the class declaration (identity labels)
###
• Inside the class declaration (methods)
